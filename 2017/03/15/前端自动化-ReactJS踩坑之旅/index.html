前端自动化-ReactJS踩坑之旅

发表于 2017-03-15 | | 本文总阅读量

如何畅快的开发前端

最近这两天在实现前端自动化的过程中踩了不少的坑，先写一篇文章压压惊。

工具

  • ReatcJS：Facebook出的前端框架，模块化设计的思想，ES5,ES6都有写法可以参考 orz
  • Redux： react更多的是扮演着MVC中的view的角色，Model以及Controll可以用flux或者更进一步用Redux，戳这个链接 里面的图片描述的很清楚
  • babel： 转换ES6以及JSX的语法到ES5，commonJS
  • webpack： 浏览器不支持commonJS，需要用到webpack或者Browserify，我选了前者，毕竟现在这个开源社区很火，坑还是多踩踩更健康
  • gulp: 前端开发自动化的利器，大有超过grunt的趋势，具有丰富的插件

以上每一个工具根据使用的插件都可以实现多类功能，这里只是使用了一小部分。

功能

废话不多说，首先上代码，git仓库，reactES6分支

先看gulpfile.js文件，用gulp都实现了哪些功能：

  1. 服务器的启动
  2. 调用babel转译JSX
  3. 调用webpack打包文件和require的依赖，实现浏览器端读取commonJS
  4. 代码的uglify，部署的时候缩小代码包的尺寸
  5. 监视代码，文件改动之后，自动实现转译以及服务器的重启

然后就终于可以安心写react了，写完之后只要command+R刷新页面就可以看到效果了，省去了一堆代码中间转译以及服务器重启的过程
babel和webpack都可以单独配置使用，更可以集成到gulp里面。

实现

还在思考怎么简略地描述清楚

一些常见的坑：
react：component的名字要大写开头
redux： reducer下面每个case里面返回的对象每一次都需要新建，可以用object.assign(用在单个对象添加属性可以返回新的对象)，用filter也会新建对象(可以用在数组上面)，
redux如何获取服务器端数据库的数据，在action里面写restful的请求给服务器端，需要配置多个action，一个action1发送请求，在action1获得请求的数据处理中dispatch一个数据完成的action
